# Free API Quickstart

## What This Adds

The project now includes an optional API-based baseline:

- `B3`: dialogue -> slots using an OpenAI-compatible API
- requirements are still generated by the deterministic slot-to-requirements stage

This is separate from the current offline baselines:

- `B1`: weak lexical rules
- `B2`: stronger normalized rule baseline

## Important Reality Check

"Free API" is usually unstable. Free tiers change, quotas disappear, and public endpoints can become rate-limited or paid later.

For a student paper, the safest practical options are:

- a local model server with an OpenAI-compatible endpoint
- a hosted endpoint that currently offers a free tier

The code does not hardcode one provider. You can point it at any OpenAI-compatible endpoint.

## Environment Variables

Set these before running the API baseline:

```bash
export REQ_LLM_BASE_URL="http://localhost:11434/v1"
export REQ_LLM_MODEL="your-model-name"
export REQ_LLM_API_KEY=""
```

Notes:

- `REQ_LLM_API_KEY` is optional if the endpoint does not require one.
- `REQ_LLM_BASE_URL` can be either:
  - a base URL ending in `/v1`
  - a full chat completions URL ending in `/chat/completions`

## Dry Run

This shows the prompt without making any API call:

```bash
python3 scripts/generate_b3_openai_compatible_slots.py --dry-run --max-samples 1
```

## Run The API Baseline

```bash
python3 scripts/run_b3_api_pipeline.py
```

This produces:

- clean slot predictions in `outputs/b3_openai_compatible_slots/`
- noisy slot predictions in `outputs/b3_openai_compatible_slots_noisy/`
- clean requirements in `outputs/b3_openai_compatible_requirements/`
- noisy requirements in `outputs/b3_openai_compatible_requirements_noisy/`
- readable input/output reports:
  - `outputs/input_output_results_b3_clean.md`
  - `outputs/input_output_results_b3_noisy.md`

## Why This Design Is Better

- You can compare a real AI baseline against `B1` and `B2`.
- The evaluation scripts stay the same.
- You are not forced into a paid provider.
- You can swap providers without rewriting the project pipeline.

## Methodological Warning

If you use a free hosted endpoint in the paper:

- record the exact provider, model name, and date used
- record whether outputs were deterministic or variable
- avoid comparing costs unless you have stable pricing evidence
- keep `B1` and `B2` as non-API baselines so your experiments remain runnable later
